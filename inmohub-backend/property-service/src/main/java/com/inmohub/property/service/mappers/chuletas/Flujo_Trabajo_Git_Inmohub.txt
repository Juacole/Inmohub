📝 Chuleta Git: Flujo por Funcionalidad (Feature Branch)

1. Preparación (Antes de empezar)
Antes de crear una rama, asegúrate de que tu base esté actualizada.
bash

	git checkout main          # Vuelve a la rama principal
	git pull origin main       # Trae los últimos cambios de la nube

Usa el código con precaución.
	
2. Crear nueva funcionalidad
No crees carpetas. Crea una rama que describa la tarea (ej: auth, login, api-fix).
bash

	git checkout -b feat-nombre-tarea   # Crea y salta a la rama
	git branch -d <nombre-rama>         # Eliminar rama

Usa el código con precaución.

3. Ciclo de Trabajo (Dentro de la rama)
Trabaja en tus carpetas (inmohub-backend, etc.). Cuando termines una parte lógica:
bash

	git status                         # Mira qué archivos has tocado
	git add .                          # Prepara todos los cambios
	git commit -m "tipo: descripción"  # Guarda el progreso localmente

Usa el código con precaución.

    Tip: Usa prefijos como api:, ui: o gateway: para saber qué parte del monorepo tocaste.

4. Publicar y Guardar (En la nube)
La primera vez que subas la rama:

	git push -u origin feat-nombre-tarea

Usa el código con precaución.

5. Finalizar y Unir (Merge)
Cuando la funcionalidad esté lista y probada:

	git checkout main                  # Vuelves a la rama principal
	git merge feat-nombre-tarea        # Traes los cambios de la rama a main
	git push origin main               # Subes todo a la nube

Usa el código con precaución.

6. Limpieza
Una vez fusionada, la rama ya no es necesaria:

	git branch -d feat-nombre-tarea    # Borra la rama local

Usa el código con precaución.

🚨 Comandos de Emergencia

    "¿En qué rama estoy?": git branch (la que tiene el * es la actual).
    "Me equivoqué de nombre de rama": git branch -m nuevo-nombre.
    "Quiero deshacer mis cambios locales (sin commit)": git restore ..
    "Tengo que cambiar de rama pero no quiero hacer commit aún":
        git stash (guarda los cambios en un "cajón" temporal).
        git checkout otra-rama (haces lo que tengas que hacer).
        git checkout rama-original y luego git stash pop (recuperas tus cambios).
